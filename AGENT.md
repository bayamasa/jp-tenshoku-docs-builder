## Git 運用ルール

mainブランチで作業中にファイル差分が発生した場合、以下の手順で進めること：

1. **ブランチ作成**: `feature/xxx` や `fix/xxx` など適切な名前でブランチを切る
2. **コミット**: 変更内容をコミットする
3. **プッシュ**: リモートにプッシュする
4. **PR作成**: GitHub上でPRを作成する
5. **マージ**: PRをマージする
6. **mainを最新化**: ローカルのmainブランチに戻り `git pull` で最新化する

直接mainブランチにコミット・プッシュしないこと。
